# InvertedPendulum Horizon-to-Failure World Model Homework

This homework evaluates a learned dynamics model directly, without policy
training, reward prediction, actor-critic updates, or custom MuJoCo XML. The
environment is Gymnasium's official `InvertedPendulum-v5`.

Students receive real MuJoCo state/action windows and train one world model:

```text
s_hat[t+1] = f_theta(s_hat[t], a[t])
```

The model sees 5 ground-truth warm-up states, then predicts 100 steps
autoregressively. The main score is **H80**: the largest prediction horizon at
which at least 80% of test windows have not failed.

## Why This Assignment

World models are often used to help policies, but policy return can hide whether
the model actually predicts physical dynamics. Here MuJoCo is the ground-truth
physics simulator. The learned model is judged by how long it can stay close to
MuJoCo under the same action sequence.

```text
MuJoCo trajectory:      s_0:T, a_0:T-1
Learned world model:   s_0:5 warmup -> s_hat_6:105 open-loop
Official metric:       horizon-to-failure survival
```

## Assignment Boundary

Students edit only:

```text
student/model.py
student/rollout.py
student/losses.py
student/metrics.py
configs/student.yaml
```

Students do not edit:

```text
wm_hw/env.py
wm_hw/dataset.py
wm_hw/eval_horizon.py
official thresholds
hidden seeds
baseline model
```

There is no controller submission. The action sequences are generated by locked
staff code using a fixed stabilizing feedback term plus smooth noise.

## Data

Each evaluation window has:

```text
states:  [N, 106, 4]
actions: [N, 105, 1]
```

State order:

```text
[cart position x, pole angle theta, cart velocity x_dot, pole angular velocity theta_dot]
```

Warm-up and rollout indexing:

```text
warmup: t = 0..4
current = s_5
h = 1 uses a_5   -> predicts s_hat_6
h = 100 uses a_104 -> predicts s_hat_105
```

Only finite windows with `abs(theta_true) < 0.20` for the full window are kept.

## Quick Start

Install dependencies:

```bash
python -m pip install -r requirements.txt
```

Run tests:

```bash
pytest -q
```

Generate a smoke dataset:

```bash
python -m wm_hw.dataset \
  --config configs/colab.yaml \
  --output-dir data/public \
  --smoke
```

Train baseline:

```bash
python -m wm_hw.train \
  --config configs/baseline.yaml \
  --model baseline \
  --dataset-dir data/public \
  --output-dir artifacts/baseline \
  --smoke
```

Train student model:

```bash
python -m wm_hw.train \
  --config configs/student.yaml \
  --model student \
  --dataset-dir data/public \
  --output-dir artifacts/student \
  --smoke
```

Evaluate:

```bash
python -m wm_hw.eval_horizon \
  --checkpoint-dir artifacts/student/best_checkpoint \
  --dataset-dir data/public \
  --split test \
  --output-dir artifacts/student/eval_test

python -m wm_hw.eval_horizon \
  --checkpoint-dir artifacts/student/best_checkpoint \
  --dataset-dir data/public \
  --split ood \
  --output-dir artifacts/student/eval_ood
```

Plot:

```bash
python -m wm_hw.plotting \
  --eval-dir artifacts/student/eval_test \
  --output-dir artifacts/student/plots
```

## Metrics

Primary metric:

```text
H80 = max h such that at least 80% of windows survive to step h
```

Failure requires two consecutive violated prediction steps. Default thresholds:

```text
angle_error_rad = 0.075
cart_pos_error_m = 0.10
cart_vel_error_mps = 0.75
pole_vel_error_radps = 1.00
consecutive_fail_steps = 2
```

Auxiliary metrics:

```text
H50
mean_survival_steps
median_survival_steps
success_rate@5/@10/@25/@50/@100
one_step_rmse
open_loop_rmse@100
```
