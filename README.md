# Homework: InvertedPendulum World Model Scoreboard

## Objective

Train a single world model for the MuJoCo `InvertedPendulum-v5` environment and
evaluate how long it can predict future states before drifting away from MuJoCo
ground truth.

There is no policy learning, no reward prediction, no actor-critic, and no
model-based planning. The task is only dynamics prediction:

```text
s_hat[t+1] = f_theta(s_hat[t], a[t])
```

You are given a runnable starter implementation. Your goal is to improve its
long-horizon open-loop prediction performance.

## How You Will Run It

The official student workflow is through Colab:

```text
notebooks/homework_colab.ipynb
```

The notebook handles setup, dependency installation, MuJoCo data generation,
starter training, official evaluation, plotting, and artifact packaging.

You should not need to type the Python commands manually during normal use. The
CLI commands near the end of this README are only a local/debugging reference
for students who want to reproduce the notebook outside Colab.

## Setup

The environment is:

```python
gymnasium.make("InvertedPendulum-v5")
```

The state is:

```text
s = [x, theta, x_dot, theta_dot]
```

where `x` is cart position, `theta` is pole angle, `x_dot` is cart velocity, and
`theta_dot` is pole angular velocity. The action is one cart force:

```text
a = [force]
```

Gymnasium reward is not used. MuJoCo provides the ground-truth trajectory, and
your learned world model predicts future states under the same action sequence.

## Data

The dataset contains MuJoCo state/action windows:

```text
states:  [N, warmup_steps + max_horizon + 1, 4]
actions: [N, warmup_steps + max_horizon, 1]
```

For the public scoreboard:

```text
warmup_steps = 10
max_horizon  = 1000
states       = [N, 1011, 4]
actions      = [N, 1010, 1]
```

Actions are generated by locked staff code using stabilizing feedback plus
smooth random noise. You do not train or submit a controller.

The public test/OOD splits are for debugging and comparison. Official grading
uses TA-private train/val/test/OOD data generated with hidden seeds.

### 3. Evaluate quantitatively

The official metrics are normalized state-space prediction errors and valid
prediction time scores.

The normalized MSE curve reports:

```text
nMSE@1
nMSE@5
nMSE@10
nMSE@90
nMSE@100
nMSE@200
nMSE@500
nMSE@1000
nMSE_AUC
```

Each state dimension is normalized by the train-split state standard deviation.
`nMSE@H` is the average normalized MSE over rollout steps `1..H`; `step_nMSE@H`
is also saved for the instantaneous error at exactly step `H`.

The primary scoreboard metric is:

```text
VPT80@0.25
```

This is the largest prediction horizon such that at least 80% of windows remain
below normalized MSE threshold `0.25`.

Additional metrics include:

```text
VPT50@0.25
VPT80@0.10
VPT80@0.50
one_step_rmse
open_loop_rmse@horizon
```

Higher VPT is better. Lower nMSE is better.
