# InvertedPendulum VPT Scoreboard World Model Homework

This homework evaluates a learned dynamics model directly. There is no policy
training, no reward prediction, no actor-critic, and no custom MuJoCo XML.
MuJoCo, through Gymnasium `InvertedPendulum-v5`, is the ground-truth physics
simulator.

Students train one world model:

```text
s_hat[t+1] = f_theta(s_hat[t], a[t])
```

The model receives 10 ground-truth warmup states, then predicts open-loop for up
to 990 steps. The primary scoreboard metric is **VPT80@0.25**: the largest
prediction horizon where at least 80% of windows remain below normalized state
MSE `0.25`.

## Assignment Boundary

Students edit only:

```text
student/model.py
student/rollout.py
student/losses.py
student/metrics.py
configs/student.yaml
```

Locked code handles MuJoCo data generation, normalization, baseline training,
official rollout, official nMSE/VPT metrics, checkpointing, and hidden grading.

## Data

State order:

```text
[cart position x, pole angle theta, cart velocity x_dot, pole angular velocity theta_dot]
```

Dataset windows are fully config-driven:

```text
states:  [N, warmup_steps + max_horizon + 1, 4]
actions: [N, warmup_steps + max_horizon, 1]
```

Public scoreboard defaults:

```text
warmup_steps = 10
max_horizon = 990
states       = [N, 1001, 4]
actions      = [N, 1000, 1]
```

Actions are generated by locked staff code using fixed stabilizing feedback plus
smooth AR(1) noise. Students do not train or submit a controller.

## Quick Start

Use non-slow tests for normal Colab setup:

```bash
python -m pip install -r requirements.txt
pytest -q -m "not slow"
```

Lightweight dev run:

```bash
python -m wm_hw.dataset --config configs/dev.yaml --output-dir data/dev --smoke
python -m wm_hw.train --config configs/baseline.yaml --model baseline --dataset-dir data/dev --output-dir artifacts/baseline --smoke

  --checkpoint-dir artifacts/student/best_checkpoint \
  --dataset-dir data/dev \
  --split test \
  --horizon auto \
  --eval-config configs/official_eval.yaml \
```

## Official Metrics

Official evaluation uses locked `wm_hw.official_rollout` and
`wm_hw.official_metrics`, not student metric code.

Reported metrics include:

```text
nMSE@1, nMSE@5, nMSE@10, nMSE@90, nMSE@100, nMSE@200, nMSE@500, nMSE@990
nMSE_AUC
VPT80@0.10, VPT80@0.25, VPT80@0.50
VPT50@0.10, VPT50@0.25, VPT50@0.50
```

`nMSE` normalizes each state dimension by the train-split observation standard
deviation stored in the checkpoint normalizer.

## Training Cost Guardrail

Long scoreboard windows can be 1000 actions long. Training does not backprop
through the full window by default. `training.train_sequence_length` samples a
short random subwindow, and `loss.rollout_train_horizon` controls short BPTT
rollout loss. Final evaluation can still run for 990 steps.
